# Preprocessing  
## PossibleYards Generator

Responsible for analyzing each row of the distance spreadsheet and identifying which columns have values **less than 379.45 meters**.  
The program records the indices of these columns and exports the result in a set-of-sets list format.

### Processing Structure
To ensure data integrity, the program automatically performs preprocessing on the spreadsheet:
1. **Removes the first row** (header/titles).
2. **Removes the first two columns** (identifiers or metadata).
3. The remaining columns are treated as possible positions for the calculation.

---

### Virtual Environment Setup and Execution

It is recommended to use a virtual environment to isolate the project dependencies:

1. **Create the virtual environment:**
   ```bash
   python -m venv venv
   .\venv\Scripts\activate
   ```

2. **Install dependencies:**
   ```bash
   pip install -r .\requirements.txt
   ```

3. **Run the preprocessing**
   ```bash
   python pre_processamento_mfs.py <input file>
   ```

# Running the Optimization in CPLEX

This section describes how to use the `possiveisPatios` file generated by the Python script as input for the optimization model solved with **IBM ILOG CPLEX**.

---

# Prerequisites

To run the optimization model it is necessary to have installed:

- IBM ILOG CPLEX Optimization Studio  
- `possiveisPatios` file generated by the Python script  
- `matrizDeDistancias.xlsx` file  

# Configuration
- Change the names of the input variables in the `covering.mod` file (according to the instance that will be optimized)
- Change the value of the `cplex.tilim` variable for the execution time

# Execution
Start running the optimizer in CPLEX and monitor the process.
